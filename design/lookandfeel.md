Overall style should be flat, light on the drop shadows.

I don't like a ton of rounded corners.

I want the site to look fun not stuffy, the colors should be bright and inviting. This is a place to be silly and have fun, not a boardroom.

Color palette - lets try something dark with really sharp color contrasts.

The typography, something almost newspaper like, in someways this is the bulletin of what people like and what they want to do.

Typography roles should be strict:

1. Sans serif owns the interface.
2. Serif owns reading copy.
3. Small meta labels also use sans serif.

Use the condensed sans for:

1. Navigation
2. Buttons
3. Form controls
4. Section headings
5. Counts, badges, chips, status labels
6. Editable titles and other clearly interactive text

Use the serif for:

1. Descriptions
2. Longer helper copy
3. Empty-state explanations
4. Candidate and bracket supporting copy when it is meant to be read, not acted on

Avoid mixing serif and sans at the same hierarchy inside one component unless they have distinct jobs:

1. Sans for the action or label
2. Serif for the explanation underneath

Use uppercase selectively, not everywhere.

Good places for uppercase:

1. Main navigation
2. Primary buttons
3. Small badges and section markers

Avoid all-caps for:

1. Longer descriptive copy
2. Most field content
3. Candidate and bracket names when normal case reads better

The accent colors should have clear jobs:

1. Yellow is the primary action color
2. Cyan/blue is for active state, winner state, focus, and selection
3. Red should be avoided unless there is a strong reason

Clickable areas should look clickable. Card hovers and active states should be obvious without making nearby non-clickable text look interactive.

Editable states should also be obvious. When a card is being edited inline, that should read clearly as an editing mode. When editing is done, the card should be able to collapse back into a calmer summary presentation.

Large forms should avoid feeling permanently open. Inline editing is fine, but the user should be able to stop editing and trust that the draft has been saved.
